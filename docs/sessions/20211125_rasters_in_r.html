Rename the layers: `fill_1` to `nir` (near-infrared), `fill_2` to `red`, `fill_3` to `green`. Plot the raster 4. Calculate the [Normalized difference vegetation index](https://en.wikipedia.org/wiki/Normalized_difference_vegetation_index) (NDVI) based on near infrared and red layers: `ndvi = (nir - red)/(nir + red)` 5. Plot `ndvi` and write it as a new GeoTIFF (*.tif) file: `ndvi_ename.tif` --- class: left, top ## INTERMEZZO: big files? No problem Big rasters? No problem! From [`raster` documentation](https://rspatial.github.io/raster/reference/raster-package.html): _The package can work with large files because the objects it creates from these files only contain information about the structure of the data, such as the number of rows and columns, the spatial extent, and the filename, but it does not attempt to read all the cell values in memory. In computations with these objects, data is processed in chunks._ This holds true for `terra` and `stars` as well, of course. --- background-image: url({{ site.baseurl}}/assets/images/background_challenge_3.png) class: left, top # Challenge 3 Calculate a canopy height raster from a digital **terrain** model (DTM) and a digital **surface** model (DSM).
