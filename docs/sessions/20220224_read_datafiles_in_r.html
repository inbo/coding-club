Example: Unicode - The characters are stored in the computer as one or more __bytes__ - The (character) encoding is a __set of mappings__ between the bytes in the computer and the characters in the character set. Without it, the data looks like garbage. Example: UTF-8 - __UTF-8__ is the most widely used way to represent Unicode text. Please, use it always - But, UTF-8 is only one of the possible ways of encoding Unicode characters: a character can actually be mapped to different byte sequences, depending on which encoding was used for the document * __Note__: this page is an extract from the article [Character encodings for beginners](https://www.w3.org/International/questions/qa-what-is-encoding) --- background-image: url({{ site.baseurl}}/assets/images/background_challenge_3.png) class: left, top # Challenge 3 Read the data file [`20220224_rainfall_waterloo.txt`](https://github.com/inbo/coding-club/blob/master/data/20220224/20220224_rainfall_waterloo.txt) containing meteorological data from Waterloo in Walloon language and using apex (') as grouping mark (e.g. 1000 -> 1'000) Be aware of these three aspects: 1.
